Cómo construí mi primera página web personal

Reflexión sobre el proceso de diseñar, programar y publicar mi primera página web como espacio para proyectos, aprendizaje e innovación.

📅 Mayo 2026 ⏱ 5 min lectura ✍️ Antonia I. Morales Gutiérrez

Contexto ◆

Siempre me ha interesado construir espacios donde pueda conectar lo que estudio, lo que diseño y lo que voy aprendiendo. La idea de crear una página web personal nació como una forma de ordenar mis proyectos, mostrar mi perfil académico y tener un lugar propio para documentar procesos relacionados con física, programación, robótica e innovación.

Más que hacer una página terminada de inmediato, quise tomar este proyecto como una primera versión: algo funcional, publicado y editable, que pudiera crecer conmigo a medida que avance en nuevos proyectos.

La idea inicial ◆

📐

Comencé diseñando a mano la estructura general: qué secciones quería incluir, cómo quería que se sintiera visualmente la página y qué tipo de información debía aparecer en el inicio, proyectos, blog, sobre mí y contacto.

✍️ Bocetar → 🎨 Imaginar estilo → 💻 Convertir a código → 🚀 Publicar

Decisiones de diseño ◆

Quería que la página se sintiera personal, pero también profesional. Por eso trabajé con una paleta de rosados, morados y tonos claros, combinada con tipografías elegantes y tarjetas suaves. La idea era que el sitio transmitiera creatividad, orden y una relación sutil con tecnología e innovación.

  • Usar una identidad visual coherente entre todas las subpáginas.
  • Separar el contenido en secciones claras: proyectos, blog, sobre mí y contacto.
  • Crear tarjetas para que cada proyecto o entrada pudiera crecer de forma ordenada.
  • Agregar modo claro y oscuro para mejorar la experiencia visual.

Herramientas que utilicé ◆

El desarrollo combinó edición manual, apoyo de inteligencia artificial y publicación con GitHub Pages. La parte principal del aprendizaje estuvo en entender cómo se relacionan los archivos, las rutas, los enlaces, las imágenes y los estilos.

🌐

HTML

Para construir la estructura de las páginas, secciones, botones, tarjetas y enlaces internos.

🎨

CSS

Para definir colores, fuentes, distribución, modo oscuro, tarjetas y detalles visuales.

⚙️

JavaScript

Para funciones simples como el cambio entre versión clara y oscura.

🤖

IA generativa

Para explorar ideas visuales, ordenar contenido, corregir código y acelerar iteraciones.

🐙

GitHub

Para organizar el repositorio, guardar cambios y mantener una versión pública del proyecto.

🚀

GitHub Pages

Para publicar la página de forma gratuita y poder acceder a ella desde internet.

Qué fui probando ◆

Durante el proceso fui ajustando muchas cosas pequeñas: rutas de imágenes, nombres de archivos, enlaces entre páginas, estructura de carpetas y detalles del diseño. Varias veces algo funcionaba localmente en VS Code, pero no se veía igual en GitHub Pages, lo que me obligó a revisar con más cuidado las mayúsculas, minúsculas y rutas relativas.

  • Probé distintas versiones del inicio hasta encontrar una estructura más clara.
  • Organicé subpáginas para que cada proyecto tuviera su propia entrada.
  • Ajusté imágenes y rutas para que funcionaran tanto localmente como en la versión publicada.
  • Fui corrigiendo el diseño para que se viera más limpio, coherente y menos compacto.

Lo que aprendí ◆

🧩

Estructura web

Comprendí mejor cómo se conectan HTML, CSS, JavaScript, carpetas, imágenes y enlaces dentro de un sitio estático.

🔍

Depuración

Aprendí a revisar errores pequeños, como nombres de archivos, rutas relativas y diferencias entre la vista local y GitHub Pages.

🚀

Publicación

Logré pasar de una idea inicial a una página funcional y publicada, entendiendo el flujo diseño → código → repositorio → web.

Próximo paso ◆

🔭

Seguir iterando la página

Quiero continuar actualizando el sitio con nuevos proyectos, entradas de blog y mejoras visuales. También me interesa explorar a futuro la posibilidad de alojar una versión desde una Raspberry Pi 5 para aprender más sobre servidores, redes y despliegue propio.
